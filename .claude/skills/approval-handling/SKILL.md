# ✅ Approval Handling Skill

**Purpose**: Manage approval workflows for high-risk operations
**Core Function**: Block → Request → Approve/Reject → Execute/Cancel
**Status Tracking**: PENDING → APPROVED/REJECTED → EXECUTED/CANCELLED

---

## 🎯 What Gets Approved

### High-Risk Operations
- 💰 Financial transactions (>$100)
- 📧 Emails to external recipients
- 🗑️ File deletions (>5 files)
- 🔐 Access permission changes
- 📝 Contract/legal documents
- 🚀 Production deployments
- 📊 Data exports
- 🔄 System configuration changes

---

## 🔄 Approval Workflow

```
1. DETECT RISK
   ↓
2. BLOCK OPERATION
   ↓
3. CREATE APPROVAL REQUEST
   Status: PENDING
   ↓
4. NOTIFY APPROVER
   ↓
5. AWAITING DECISION
   ↓
   ├─→ APPROVED → Execute → EXECUTED
   └─→ REJECTED → Cancel → CANCELLED
```

---

## 📊 Status States

### 1. 🟡 PENDING
**Meaning**: Awaiting human decision

**Properties**:
- Created timestamp
- Requester info
- Operation details
- Risk assessment
- Waiting for approval

**Actions available**:
- Approve
- Reject
- Request more info
- Escalate

---

### 2. ✅ APPROVED
**Meaning**: Human authorized execution

**Properties**:
- Approved timestamp
- Approver name
- Approval notes
- Valid for: 24 hours (expires)

**Next steps**:
- Execute operation
- Mark as EXECUTED
- Log completion

---

### 3. ❌ REJECTED
**Meaning**: Human denied execution

**Properties**:
- Rejected timestamp
- Rejector name
- Rejection reason
- Alternative suggested

**Next steps**:
- Cancel operation
- Mark as CANCELLED
- Notify requester

---

### 4. ⚙️ EXECUTED
**Meaning**: Successfully completed

**Properties**:
- Executed timestamp
- Executor (AI/Human)
- Result details
- Audit log entry

**Final state**: Operation complete

---

### 5. 🚫 CANCELLED
**Meaning**: Rejected and cancelled

**Properties**:
- Cancelled timestamp
- Cancellation reason
- Alternative taken (if any)

**Final state**: Operation aborted

---

### 6. ⏰ EXPIRED
**Meaning**: Approval time limit exceeded

**Properties**:
- Expired timestamp
- Original approval
- Requires re-approval

**Action**: Create new approval request

## 🚫 Blocking Mechanism

### How Blocking Works

**1. Pre-Execution Check**
```markdown
Before any high-risk operation:

1. Check if operation requires approval
2. If YES:
   a. Check if approval exists
   b. Check approval status
   c. Check if approved & not expired
3. If not approved: BLOCK and create approval request
4. If approved: Proceed with execution
```

**2. Block Implementation**
```markdown
Operation: send_email(recipient)

Step 1: Risk check
  - Is recipient external? YES
  - Risk level: HIGH
  - Requires approval: YES

Step 2: Approval check
  - Find approval request for this operation
  - Status: PENDING (not approved yet)

Step 3: BLOCK
  - Do NOT send email
  - Display: "⛔ BLOCKED - Awaiting approval"
  - Log block event
  - Notify requester

Step 4: Wait for approval
  - Check every 5 minutes
  - If approved: Execute
  - If rejected: Cancel
  - If expired: Create new request
```

**3. Bypass Prevention**
```markdown
Anti-bypass measures:

❌ Cannot skip approval check
❌ Cannot force execute without approval
❌ Cannot modify approval status directly
❌ Cannot extend expired approvals

✅ Only human can approve
✅ Approval must be explicit (checkbox)
✅ All blocks logged for audit
✅ Failed bypass attempts logged
```

## 📋 Approval Workflows

### Workflow 1: External Email

```markdown
1. DETECT: Email to external domain
   ↓
2. BLOCK: Stop email sending
   ↓
3. CREATE REQUEST:
   - Extract email details
   - Assess risk
   - Create approval request
   - Move to pending/
   ↓
4. NOTIFY: Alert approver via:
   - Dashboard update
   - Email notification
   - Slack message (if configured)
   ↓
5. AWAIT: Check status every 5 min
   ↓
6a. IF APPROVED:
    - Move to approved/
    - Send email
    - Log as executed
    - Move to executed/
    ↓
6b. IF REJECTED:
    - Move to rejected/
    - Cancel email
    - Log reason
    - Notify requester
```

---

### Workflow 2: Financial Transaction

```markdown
1. DETECT: Payment operation
   ↓
2. BLOCK: Prevent transaction
   ↓
3. CREATE REQUEST:
   - Payment amount
   - Recipient
   - Reason
   - Risk score
   ↓
4. ESCALATE: Route by amount
   - <$100 → Operations
   - $100-$1000 → Manager
   - >$1000 → Director
   ↓
5. MULTI-APPROVAL: If >$10k
   - Require 2 approvals
   - Manager + Director
   ↓
6. EXECUTE: After all approvals
   - Process payment
   - Log transaction
   - Archive approval
```

---

### Workflow 3: Bulk Deletion

```markdown
1. DETECT: Delete >5 files
   ↓
2. BLOCK: Prevent deletion
   ↓
3. LIST: Show what will be deleted
   - File names
   - File sizes
   - Last modified dates
   ↓
4. CONFIRM: Explicit confirmation
   - Checkbox for each file
   - Type "DELETE" to confirm
   ↓
5. APPROVE: Human reviews list
   ↓
6. EXECUTE: Delete approved files only
   - Skip any not checked
   - Log each deletion
   - Cannot undo
```

## 📊 Status Tracking

### Status Transitions

**Valid transitions:**
```
PENDING → APPROVED → EXECUTED ✓
PENDING → REJECTED → CANCELLED ✓
PENDING → EXPIRED → PENDING (new request) ✓
APPROVED → EXPIRED → PENDING (re-approval) ✓

Invalid transitions:
PENDING → EXECUTED ✗ (must approve first)
REJECTED → EXECUTED ✗ (cannot execute rejected)
EXECUTED → PENDING ✗ (cannot undo)
```

## ⏰ Expiration Handling

### Expiration Rules

**Default expiration**: 24 hours from creation

**Why expire?**
- Prevent stale approvals
- Context may change
- Security best practice
- Force re-review for delayed operations

**Expiration process:**
```markdown
Every hour:
1. Check all APPROVED requests
2. If approved_timestamp + 24h < now:
   a. Mark as EXPIRED
   b. Move to expired/
   c. Notify approver
   d. Block re-execution

To execute expired approval:
1. Create NEW approval request
2. Reference expired request
3. Request fresh approval
4. Proceed if approved
```

## 🎯 Best Practices

### DO ✅

1. **Review promptly**
   - Check dashboard daily
   - Respond to CRITICAL within 1h
   - Clear PENDING backlog

2. **Understand before approving**
   - Read full context
   - Check risk assessment
   - Verify details accurate

3. **Document decisions**
   - Add notes to approvals
   - Explain rejections
   - Suggest alternatives

4. **Monitor patterns**
   - Track approval rates
   - Identify bottlenecks
   - Adjust thresholds

5. **Audit regularly**
   - Review approval logs
   - Check for anomalies
   - Verify policy compliance

---

### DON'T ❌

1. **Don't rubber-stamp**
   - Actually review requests
   - Don't approve blindly
   - Risk exists for a reason

2. **Don't share approval access**
   - One person per approval
   - No shared accounts
   - Maintain accountability

3. **Don't bypass system**
   - Follow approval process
   - No "just this once" shortcuts
   - Bypasses undermine security

4. **Don't ignore CRITICAL**
   - CRITICAL means urgent
   - Don't delay indefinitely
   - Respond or escalate

5. **Don't forget to check expiration**
   - Expired approvals invalid
   - Re-review before executing
   - Old approval ≠ current approval
